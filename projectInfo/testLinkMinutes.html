    • Once tests are imported from Cartman to Kenny, how do you continue to bind results to that testcase.
    • Proposal: Cartman TestCaseVersion1 gets imported into Kenny TestCaseVersion1; TestCaseVersion1 is executed, results are generated, notes are added. Meeting with GoodInfo QA: 06/03/2003

Meeting Goals

  • Information Exchange and TestLink Demo

Meeting Notes

  • Attendees: Erik Spencer, Mikhail Granovskiy, Darren Merritt
  • Initial meeting to discuss GoodInfo team usage of TestLink.
    • Discussed TestLink background
    • Demonstration of TestLink functionality
    • Discussed keyword functionality: Usage for outsourcing sort criteria in addition to categorization.
    • Discussed Metrics currently used and need for future types
    • Discussed GoodInfo automated tests
    • Discussed GoodInfo current test document structure
  • Comments/Feedback:
    • Metrics: Would like to see a way to track test status for a product against current bugs. For example, during a test cycle, when testing is 50% complete, there would be a way to see what bugs have been attributed to that testing so far. In addition, bug trends across builds could be linked to test cases.
    • Metrics: Estimates on duration of a test case (initial number entered by a tester) vs. actual duration of test case completion. This will allow visibility into how long future tests will take...and how accurate QA is at setting targets for future test completion.
    • Feature Request: Add a link to bugzilla from results window. For example, user fills in notes and enters a status of failed. There could be a window that pops up asking if you want to enter a bug now?, and if so, then it pops up bugzilla in a new window. Alternatively, it could just pop up a new window with "enter new bug" in bugzilla with the project/component data already "seeded". Or it could be as simple as a link to the enter new bug page...
    • Keys: Ability to add keys "retroactively" to a testcase or group of testcases. For example: a user realizes during testing that several tests need to be included that are not. The keys could be added and the testcases made available for that cycle as part of that keyword "category"
    • GoodInfo automation tests are separate from the Word document based tests. Results from automation are generated by comparison to "baseline" output that is periodically updated when baseline (expected result) changes. Results could be imported directly into results section of TestLink.

Action Items

  • Darren: Schedule Meeting to discuss metrics ideas/preferences. Orca QA, GoodInfo QA, TestLink Team
  • Darren: Add applicable comments/feedback to bugzilla under project TestLink
  • Darren: Work with GoodInfo QA team to investigate automated tests/results in detail.
  • Darren: Work with GoodInfo QA team to determine best solution for Word document data extraction. No styles currently available in docs.
